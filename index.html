私という人間

— ABOUT
製造現場で22年、改善とチームを動かす → 不動産の実務でドメインを習得 → AIも活用しながら「動くもの」をつくる

一本の軸は、現場を知り、課題を見つけ、手を動かして改善することです。手段が工程改善からPower Query、GAS、JavaScript、OCR、AI活用へ広がっただけで、やっていることは一貫しています。

キャリアの大半は製造業です。木材プレカット加工の現場で22年以上、CAD・NC加工・検品・工程管理・在庫管理・品質管理に携わり、現在は工場長として現場マネジメントと改善を担っています。数字で判断し、ボトルネックを見つけ、カイゼンを回し続ける——これが私の仕事の原点です。

副業では賃貸不動産の実務に入り、物件情報の確認、入力、解約受付、電話取次ぎなど、現場で発生する確認・転記・判断の粒度を学んでいます。ここで得た知見を、物件データ整形ツールや入力支援MVPに横展開しています。

「要件を完璧に固めてから作る」のではなく、「動く試作を見せ、現場と対話しながら要件を確かめる」。製造業の試作・改善文化と地続きの、現場課題を整理し、動く試作に落とす進め方です。

03

OCR / LLM 伝票読取ツール

製造業ドメイン / 受入・検品の入力自動化 / OCR・LLMデモ
課題 / ISSUE
受入・検品で発生する紙の納品伝票を、手作業で在庫・発注の入力へ転記しており、入力負担と転記ミスが残っていた。
打ち手 / ACTION
伝票画像をOCR/LLMで読み取り、品名・数量・単価・伝票番号を構造化データに変換。金額整合チェックで要確認行を提示する流れを試作。
結果 / RESULT
紙伝票の転記を、抽出→検証→CSV出力の再現可能な流れとして整理。在庫・発注の入力前処理を補助するMVPとして検証中。
OCRLLMHTMLCSV出力項目抽出受入・検品
紙伝票の入力自動化を、製造現場の在庫・発注業務に直結させる静的デモです。処理前の伝票イメージと抽出後の構造化結果を対比表示します。

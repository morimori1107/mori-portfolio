私という人間

— ABOUT
製造現場で22年、改善とチームを動かす → 不動産の実務でドメインを習得 → AIで自分で「動くもの」をつくる

一本の軸は「現場を知り、課題を見つけ、手を動かして改善する」こと。手段が工程改善からAI実装に変わっただけで、やっていることはずっと同じです。

キャリアの大半は製造業です。工場長として22年、生産工程の改善、品質管理、コスト管理、そして複数のチームと人を動かすマネジメントを実践してきました。数字で判断し、ボトルネックを見つけ、カイゼンを回し続ける——これが私の仕事の原点です。

その後、不動産賃貸管理の実務で新たなドメイン知識を習得。空室確認、入居条件、物件査定など「現場で本当に確認すべきこと」を体で覚えました。そして近年、生成AIを業務に取り込み、判定メモ生成・公的データ統合・募集文の自動作成などを自分の手で実装しています。

「要件を完璧に固めてから作る」のではなく、「動く試作を見せ、現場と対話しながら要件を確定させる」。製造業の試作・改善文化と地続きの、現場巻き込み型の開発が私のやり方です。

01

物件判定メモ自動生成アプリ

企画 / 設計 / 実装(個人開発・自走)
課題 / ISSUE
判定メモ作成に毎回30分以上。担当者ごとに観点がばらついていた。
打ち手 / ACTION
物件情報からAIが観点付き判定メモを自動生成するWebアプリを開発。保存先を切替可能に。
結果 / RESULT
メモ作成を約8割短縮(ダミー値)、観点の標準化を実現。
HTML/JSClaude APINotion DB
「観点の標準化」は、工場での作業標準化(SOP)づくりと同じ発想。属人化を仕組みで潰す。
02

公的データ統合・物件リスク可視化

データ統合設計 / 実装
課題 / ISSUE
ハザードマップ・地価・用途地域の確認が分散し、見落としリスクがあった。
打ち手 / ACTION
公的データを物件に自動紐付けし、リスクを一画面に集約。AIが注意点をコメント。
結果 / RESULT
確認漏れの削減と説明資料の作成時間短縮(ダミー値)。
外部データ統合ハザードマップClaude API
「見落としを仕組みで防ぐ」は製造業の品質管理(ポカヨケ)の考え方そのもの。
03

OCR連携 査定・募集文自動生成

パイプライン設計 / 実装
課題 / ISSUE
紙・PDF資料の転記が手作業。募集文の品質も担当者依存だった。
打ち手 / ACTION
OCRで読み取り、差分検知と募集コピーをAIが自動生成するパイプラインを構築。
結果 / RESULT
転記工数の削減と募集文の品質均一化(ダミー値)。
Google Vision OCRClaude API差分検知
ツール選定で詰まった際、外部連携ツールから自己完結型へ即座に方針転換。手段に固執せず目的を取る判断。

開発スタイル： 1つのMVPを5×5〜10回ほど、計20回以上作り直して磨き込むのが常。速攻で動くものを出し、回しながら完成に近づける高速反復が得意分野です。
